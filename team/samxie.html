Order food: order

Orders food.

  • If index is given, will order the food at the specified index.

  • If no index is given, a food will be chosen for you. The food chosen is determined to be both desirable yet, not too deterministic that it will always be ordered.

  • This command will not order foods which you have allergies for.

  • Ordering may take some time as email and phone call is prepared.

Food is ordered by:

  1. Emailing the restaurant of the associated food for a request, and summary of the order (for recording purposes).

  2. Sending a request to our server which initiates a phone call to food’s phone number of the same message.

    Make sure that the food's contact details are correct, specifically, the phone is a Singaporean number.

Format: order [INDEX] or order

Example:

  • order 2 orders the second food listed

  • order selects a food from the list to be ordered for you

order command feature

What

The order command is the core of the application. It automates the food ordering process. It can automate the food selection process, to minimise decision making strain, while also sending the order to the restaurant, reducing the time required to order food.

Why

Programmers often ignore their appetite amidst long coding sessions, leaving their stomach rumbling. It is often too much effort to switch out of the flow state to begin the laborious process of ordering food. HackEat changes this. With a simple order command, the whole process, food selection and ordering, is streamlined.

How

Overview Flow chart

  1. Order command searches through existing Foods in the HackEat book

    1. Within each Food, a score is derived based on number of factors such as: rating, price and allergies.

  2. Foods will be ordered on a probabilistic basis, making sure a variety when ordering food often.

  3. The program will email the restaurant with a record and summary of the request for food.

  4. A phone call will be held to request food.

    1. A message generated by the client will be sent to our server. Server source code can be found here: https://github.com/CS2103JAN2018-W10-B3/hackeat-server

    2. The server will initiate a phone call using a calling service to the chosen food’s phone number.

    3. A text to voice service will be used to translate text to voice to communicate with the restaurant on the other end and order a food with a pre-canned message.

It is then assumed that the restaurant will acknowledge the order and deliver the food to the user’s address. The user will need to pay once it has arrived.

Current implementations

Food Selector

  • Foods are given scores based on ratings, price and allergy.

    • A high rating will result in a higher score

    • A low price will result in a higher score

    • Any allergies that the food matches with the user will immediately give the food a score of 0, eliminating it from the selection process

  • Foods are then given a number, based on their positions, occupying a range equivalent to their score (see diagram).

  • A random number is selected between 0 and the cumulative score (max running score)

  • The random number will land on a food (shown in diagram) and be selected.

  • This will select a food probabilistically, based on the scores. A higher score will have a higher chance of selection.

Order Manager

  • Sets up a session and generates access to email server

  • Emails restaurant the order details, an order record and summary.

  • Initiates a phone call to the food’s phone number

    • v1.4 has implemented this on client side via Twilio, but was changed due to current rejection of the use of the library.

    • Therefore the phone calling was delegated to a remote server, moved out of the client.

    • The remote server accepts a HTTP POST request with the message details, and create a phone call based on the message.

Design considerations

  • Email was used to avoid importing a large library and can be used as a record summary for the restaurant.

  • Voice call for ordering food was also chosen as it is Universal to all restaurants that deliver food. Some restaurants do nto have Uber Eats or Food Panda service.

  • A server was used to handle the phone call to keep the client lightweight, and to avoid any library import complications.

Alternatives and future considerations

Existing food ordering platform API

A promising alternative is to use an existing API from UberEats or other food ordering services to allow more services added such as: delivery time tracking, more restaurants. This would guarantee the service more reliably.

Smarter ordering AI agent

Possibly implement smarter speech with <Gather> and <Redirect> (Twilio example).

  1. <Gather> gathers data from user in form of speech, and posts to the url /create/:id/response at the REST API.

  2. API parses the information and generates new XML data specific to the response at /order/:id/response.

  3. <Redirect> redirects chat flow to /order/:id/response and retrieves XML data and confirms, repeats or declines the order.
